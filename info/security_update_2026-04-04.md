# Security Update Summary

**Date:** 2026-04-04
**Author:** Claude Code
**Scope:** Pre-publication security hardening of `octoprint_manager.py` and repo structure

---

## Why This Was Done

The repo was audited as if it were about to be published to a **public GitHub repository**.
Several secrets and vulnerabilities were found and fixed before any commit could expose them.

---

## Vulnerabilities Found & Fixed

### 1. CRITICAL — API Key Hardcoded in Source

**File:** `octoprint_manager.py` line 13
**Before:**
```python
API_KEY = "<redacted>"
```
**After:** Loaded from environment variable via `.env` file (gitignored).
```python
API_KEY = os.environ.get("OCTOPRINT_API_KEY", "")
```
**Risk if unfixed:** Anyone with read access to the repo could take full control of the printer.

---

### 2. CRITICAL — API Key Exposed in Markdown

**File:** `info/summary_2026-04-04.md`
**Before:** Real API key appeared in multiple `curl` command examples.
**After:** Replaced with `$OCTOPRINT_API_KEY` shell variable placeholder throughout.
**Risk if unfixed:** Key visible in rendered GitHub README/docs.

---

### 3. HIGH — Server IP/URL Hardcoded in Source

**File:** `octoprint_manager.py` line 12
**Before:**
```python
OCTOPRINT_URL = "http://192.168.1.192:5000"
```
**After:** Loaded from `OCTOPRINT_URL` environment variable.
**Risk if unfixed:** Exposes internal network topology; breaks portability.

---

### 4. HIGH — No `.gitignore` (secrets would have been committed)

**Created:** `.gitignore` at repo root.
**Ignores:**
- `.env` — secrets file
- `.claude/settings.local.json` — contained API keys in allowed-commands list
- `*.gcode` and `*.stl` — large binary/generated files (regenerated by scripts)
- Python cache files

---

### 5. MEDIUM — Path Traversal on File Upload

**File:** `octoprint_manager.py` — `upload_gcode()` function
**Before:** User-supplied path passed directly to `open()` with no validation.
**After:** `safe_local_path()` resolves and validates the path is inside `print_jobs/`.
**Risk if unfixed:** Attacker could upload arbitrary files from anywhere on the filesystem to OctoPrint.

---

### 6. MEDIUM — Path Traversal on Remote API Path

**File:** `octoprint_manager.py` — `select_and_print()` and `ensure_remote_folder()`
**Before:** User input injected directly into the OctoPrint API URL.
**After:** `safe_remote_path()` strips leading slashes, rejects `..`, `.`, empty components, and null bytes.
**Risk if unfixed:** Could manipulate API endpoint to access unintended OctoPrint resources.

---

### 7. LOW — Silent Exception Swallowing

**File:** `octoprint_manager.py` — `ensure_remote_folder()`
**Before:** `except Exception: pass` — all errors silently ignored.
**After:** Only HTTP 409 (folder already exists) is suppressed; all other errors are re-raised.
**Risk if unfixed:** Real failures (auth errors, network issues) hidden from the user.

---

### 8. LOW — No Request Timeouts

**File:** `octoprint_manager.py` — all `requests` calls
**Before:** No timeout set — could hang indefinitely if printer goes offline.
**After:** `timeout=15` on API calls, `timeout=60` on file uploads.

---

## New Files Added

| File | Purpose |
| --- | --- |
| `.env` | Stores real secrets — **gitignored, never commit** |
| `.env.example` | Safe placeholder template — commit this instead |
| `.gitignore` | Prevents secrets and large files from being committed |

---

## How to Configure After a Fresh Clone

```bash
cp .env.example .env
# Edit .env and set:
#   OCTOPRINT_URL=http://your-printer-ip:5000
#   OCTOPRINT_API_KEY=your_key_from_octoprint_settings

pip3 install requests python-dotenv numpy-stl
python3 octoprint_manager.py
```
